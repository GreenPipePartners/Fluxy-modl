# Contributing To Fluxy Java

Issues and design discussions are welcome. External code contributions are not accepted until Green Pipe Partners publishes and operates its contribution agreement process.

## License Boundaries

- Authored source in this repository is MPL-2.0.
- The Gradle wrapper retains its Apache-2.0 license.
- Historical WebDev-derived portions retain the MIT notice in `WEBDEV_MIT_NOTICE`.
- Future contributions require sufficient copyright and patent grants to Green Pipe Partners, LLC to maintain and distribute Fluxy Java.

## CLA Requirement

Do not submit or merge external code until Green Pipe Partners has recorded an approved agreement for the contributor. Contributors acting within employment or on behalf of an organization also require authorized employer or corporate coverage.

Company founders, employees, and contractors must also have a continuing written agreement covering contributions created after any one-time assignment's effective date.

## Contribution Rules

- Identify third-party code, generated code, and sources in the pull request.
- Do not submit material you lack authority to license.
- Preserve SPDX, copyright, license, and notice files.
- Never submit API credentials, signing material, activation credentials, customer data, or proprietary Ignition resources.
- Add tests for behavior changes and update the applicable MIT or MPL documentation.
- Use Developer Certificate of Origin sign-off only if Green Pipe formally enables it; DCO sign-off does not replace the required CLA.

See `LICENSE`, `THIRD_PARTY_NOTICES.md`, `SECURITY.md`, and `SUPPORT.md` before contributing.
